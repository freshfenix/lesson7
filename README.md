# 📘 CSS Layout Demo (Display, Flex, Grid, Position)

## 📌 Опис

Цей проєкт демонструє основні властивості CSS, які використовуються для побудови макетів сторінок:

- `display` (inline, block, inline-block)
- `flex`
- `grid`
- `float`
- `overflow`
- `position` (static, relative, absolute, fixed, sticky)

Мета — наочно показати різницю між підходами та їх поведінку в реальних умовах.

## 🎯 Основні демонстрації

### 🔹 Display

Показує різницю між:

- inline
- block
- inline-block

📌 Особливості:

- inline не підтримує `width` / `height`
- block займає всю ширину
- inline-block комбінує обидва підходи

---

### 🔹 Flexbox

Демонструє:

- вирівнювання елементів
- розподіл простору (`flex: 1`, `flex: 2`)
- зміну порядку (`order`)

📌 Використовується для одновимірних макетів (рядок або колонка)

---

### 🔹 Grid

Демонструє:

- двовимірну сітку
- об'єднання колонок (`grid-column`)
- об'єднання рядків (`grid-row`)

📌 Підходить для складних макетів

---

### 🔹 Float

Показує:

- обтікання текстом
- вирівнювання зображень
- використання clearfix

📌 Застарілий підхід, але важливий для розуміння

---

### 🔹 Overflow

Демонструє:

- `hidden`
- `scroll`
- `auto`

📌 Контроль переповнення контенту

---

### 🔹 Position

Показує всі типи позиціонування:

- `static` — за замовчуванням
- `relative` — зміщення відносно себе
- `absolute` — позиціонування відносно батька
- `fixed` — фіксація відносно viewport
- `sticky` — прилипає при скролі

---

## 🔥 Fixed vs Sticky

| Властивість | Поведінка           |
| ----------- | ------------------- |
| fixed       | завжди на екрані    |
| sticky      | прилипає при скролі |

📌 Важливо:

- `sticky` працює тільки в межах контейнера
- потрібен `top`, `bottom`, `left` або `right`
